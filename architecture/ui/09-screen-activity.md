# 09 — Screen: Activity

The accountability surface: every run, refusal, hold, conflict, advice event, and trash move the engine ever performed, in the engine's own words ([../core/08-observability.md](../core/08-observability.md)). If a user ever asks "what did Aetherloom just do?", this screen must answer completely. 🔁 Reshape of `Views/ActivityView.swift` (today: flat sample list).

## 1. Layout

```text
PageHeader "Activity" / "Everything Aetherloom does, as it happens."
Filter bar:
  [category chips: All · Sync · Safety · Conflicts · Suggestions · Providers · Errors]
  [sync set picker: All Sets ▾]   [search field (message/path contains)]
Feed (LazyVStack, newest first), grouped by run:
┌ RunGroup header: sync set name · started → finished relative · outcome badge
│   ActivityRow: time · category glyph (toned) · message · location mark? · PathText?
│   └ expandable detail (entry.detail, monospaced)
└ … ungrouped entries (no runID) render standalone
[Load More] pagination (query limit 200, offset by last date)
```

## 2. Data & behavior

| Element | Source | Status |
| --- | --- | --- |
| Feed | `session.activity(matching: ActivityQuery(...))` — the orchestrator's real `ActivityStore`; live prepend via `.activityAppended` | ✅ |
| Category filter | `ActivityQuery.categories` (server-side, i.e. store-side) | ✅ |
| Sync set filter | `ActivityQuery.syncSetID` | ✅ |
| Search | client-side substring over the fetched window; caption notes "Searching recent activity" | ✅ (windowed) |
| Run grouping | `runGroups(...)` [04 §6] | ✅ |
| Outcome badge | `RunDigest.outcome` → completed `healthy` · held/refused `paused` · stoppedForReplan/failed `attention` | ✅ |
| Row → conflict deep link | `relatedConflictID` → Conflicts screen | ✅ |
| Row context menu: Copy Message / Copy Path | pasteboard | ✅ |
| Export… | toolbar item, disabled + `PlaceholderChip("Log export coming soon")` | 🎭 |

Deep-link entry: `AppModel.show(.activity, filteredToRun: runID)` (from `RunResultToast` and sync set history) preselects the run's group and clears other filters.

## 3. Presentation rules

- Messages render verbatim — the `ActivityMessageCatalog` already speaks user language ("Moved … to trash", "Sync paused because…"). The UI adds *only* layout, relative time, and glyphs.
- Advisory entries always visibly attributed (the entry `detail` carries "Generated by … via …"); the `sparkles` glyph marks them.
- Safety entries (refusals, holds, recovery) get the shield glyph and a subtle left border in tone color — scannable but not alarming.
- Timestamps: relative in the row, absolute on hover; day separators for older content.

## 4. States

- Empty (fresh reset): `EmptyStateView` "Activity appears here the moment Aetherloom starts working."
- Filter with no matches: "Nothing matches these filters." + [Clear Filters].
- The feed never blocks on a running sync — appends stream in live. ✅

## 5. Acceptance criteria

- After demo bootstrap, the feed shows the converging pass runs, then the scripted scan: stage entries, hold and refusal entries, conflict entry with working deep link, advisory entry with attribution.
- Approving Documents in [07] produces a complete run group: run started → stages → per-operation entries (including trash causality) → run finished; toast deep-link lands on it filtered.
- "Simulate Interrupted Run" + scan surfaces the recovery entry ("reconciled … operations for unfinished run") in the Safety category. ✅
- Category chips and set picker round-trip through `ActivityQuery` (bridge test), not client filtering, except search.
